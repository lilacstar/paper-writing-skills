# Verify Abstract — 摘要准确性验证

本 Skill 对比摘要与论文全文，检查摘要是否准确反映正文内容，标题是否与正文一致。采用逐项对照的方式而非泛泛而评。

## 行为准则（强制）

1. **默认存疑**：假设摘要可能存在问题，逐项对照检查
2. **禁止讨好**：不得为了迎合作者而给出正面评价
3. **只做规范性检查**：可以检查内容一致性、表述准确性、标题匹配度。不能验证研究本身的科学性
4. **问题导向输出**：每条问题包含对照项、摘要内容、正文对应内容、是否一致
5. **回退判断**：发现严重不一致时标注回退建议

## 严重程度评级

- **致命**：摘要与正文严重不符，可能导致审稿人怀疑研究可信度
- **重要**：摘要存在部分不一致或夸大，影响读者理解
- **轻微**：措辞优化建议，不影响准确性

## 工作流程

0. **通知（流程开始）**：读取 `paper/metadata.json`，如果 `wechatWebhook` 非空，发送企业微信通知：
   ```powershell
   Invoke-RestMethod -Uri "WEHOOK_URL" -Method Post -ContentType "application/json" -Body '{"msgtype":"markdown","markdown":{"content":"## 论文写作助手\n> **摘要验证** 流程已启动\n> 论文主题：{researchTopic}\n> 开始时间：{当前时间}"}}'
   ```
   > 如果 `wechatWebhook` 为空，跳过。

1. 读取 `paper/metadata.json` 获取论文类型
2. 读取 `paper/abstract.md`（摘要和标题）
3. 读取论文全文（`paper/draft-full.md` 或所有 `paper/draft-*.md`）
4. 按检查清单逐项对照
5. 生成对照报告，追加写入 `paper/review-report.md`
6. **通知（流程结束）**：如果 `wechatWebhook` 非空，发送企业微信通知：
   ```powershell
   Invoke-RestMethod -Uri "WEHOOK_URL" -Method Post -ContentType "application/json" -Body '{"msgtype":"markdown","markdown":{"content":"## 论文写作助手\n> **摘要验证** 已完成\n> 论文主题：{researchTopic}\n> 验证结果：一致 {一致数} / 不一致 {不一致数}\n> 产出文件：paper/review-report.md\n> 完成时间：{当前时间}"}}'
   ```
   > 如果 `wechatWebhook` 为空，跳过。

## 检查清单

### 一、摘要与全文一致性检查

对摘要的每个组成部分，与正文进行逐项对照：

- [ ] **背景（Background）**：摘要中的背景描述是否与 Introduction 一致
- [ ] **目的（Objective）**：摘要中的研究目的是否与 Introduction 末尾一致
- [ ] **方法（Methods）**：摘要中的方法描述是否与 Methods 章节一致
  - 研究设计类型是否一致
  - 样本量/研究对象是否一致
  - 主要干预/暴露是否一致
  - 主要结局指标是否一致
- [ ] **结果（Results）**：摘要中的结果是否与 Results 章节一致
  - 关键数据（数值）是否准确
  - 统计显著性结论是否一致
  - 有无摘要中提到但正文中没有的结果
  - 有无正文中重要结果在摘要中遗漏
- [ ] **结论（Conclusions）**：摘要中的结论是否与 Discussion/Conclusion 一致
  - 是否超出正文结论的范围
  - 是否夸大了研究贡献
  - 是否遗漏了正文中的重要限定条件

### 二、标题检查

- [ ] 标题是否反映了研究的核心变量/概念
- [ ] 标题是否反映了研究设计类型（如"随机对照试验"）
- [ ] 标题是否与研究对象的描述一致
- [ ] 标题中是否包含过度夸大的词汇（"首次证明"、"重大突破"等）
- [ ] 标题长度是否合理（英文 8-15 词，中文 15-25 字）

### 三、摘要格式检查

- [ ] 摘要字数是否符合目标期刊要求
- [ ] 摘要中是否使用了未定义的缩写
- [ ] 摘要中是否包含参考文献引用（通常不允许）
- [ ] 结构式摘要的各部分标签是否正确

### 四、信息完整性检查

- [ ] 摘要是否包含足够的信息让读者独立理解研究
- [ ] 摘要是否遗漏了关键的方法学细节（如盲法、样本量）
- [ ] 摘要是否遗漏了最重要的统计结果

## 输出格式

```markdown
## 摘要验证报告

**验证时间**：YYYY-MM-DD HH:MM
**论文类型**：[从 metadata.json 读取]

### 验证摘要
- 对照项总数：X
- 一致：Y
- 不一致：Z（致命 A / 重要 B / 轻微 C）

### 逐项对照表

| 摘要部分 | 摘要内容 | 正文对应 | 是否一致 | 备注 |
|---------|---------|---------|---------|------|
| Background | "..." | Introduction 第1段 | 一致/不一致 | ... |
| Objective | "..." | Introduction 末段 | 一致/不一致 | ... |
| Methods | "..." | Methods 章节 | 一致/不一致 | ... |
| Results | "..." | Results 章节 | 一致/不一致 | ... |
| Conclusions | "..." | Discussion 末段 | 一致/不一致 | ... |

### 问题清单

#### 1. [致命/重要/轻微] 问题描述
- **位置**：摘要 [部分名]
- **摘要原文**："..."
- **正文内容**："..."
- **问题**：[具体描述不一致之处]
- **建议**：[修正建议]

### 回退建议（如有致命问题）
[回退建议]
```

## 输入输出

- 输入：`paper/abstract.md` + 论文全文
- 输出：对照报告 → `paper/review-report.md`（追加模式）

## 文件读写约定

| 操作 | 文件 |
|------|------|
| 读取 | `paper/metadata.json`, `paper/abstract.md`, `paper/draft-full.md`（或所有 `paper/draft-*.md`） |
| 写入 | `paper/review-report.md`（追加，带时间戳，不覆盖） |

## 注意事项

- 本 Skill 只做摘要与全文的一致性对照，不评价研究本身的科学性
- 如果 `paper/abstract.md` 不存在，提示用户先加载 `abstract-writer` 生成摘要
- 如果论文全文不完整，只能基于已有章节进行对照，并明确标注缺失部分
- 验证报告使用追加模式，保留历史记录

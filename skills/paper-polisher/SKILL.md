# Paper Polisher — 语言润色与格式规范

本 Skill 负责论文的语言润色和格式规范检查。根据论文类型自动切换中英文模式。

## 前置条件

确认 `paper/metadata.json` 存在。确认待润色的章节文件存在。

## 工作流程

1. 读取 `paper/metadata.json` 获取论文类型和语言
2. 确认润色范围：
   - 全文润色：读取所有 `paper/draft-*.md`
   - 单章节润色：读取指定章节文件
   - 摘要润色：读取 `paper/abstract.md`
3. 加载对应的语言规范参考资料
4. 执行润色检查，生成修改建议
5. 根据用户确认，更新对应文件

## 语言润色规范

### 通用原则

- 保持作者原意，不改变核心论点和数据
- 优先建议修改，而非直接替换（让用户决定是否采纳）
- 标注每处修改的理由
- 区分"必须修改"和"建议修改"

### 英文润色检查项

| 检查类别 | 具体内容 |
|---------|--------|
| 语法 | 主谓一致、时态一致、冠词使用 |
| 学术表达 | 避免口语化（a lot of → numerous/substantial）、避免缩写（don't → do not） |
| 时态规范 | Methods/Results 用过去时，Introduction/Discussion 现在时为主 |
| 语态 | 避免过度使用被动语态（当主动更清晰时） |
| 冗余 | 去除无意义的填充词（it should be noted that、it is important to mention） |
| 段落衔接 | 检查段落间的逻辑过渡，避免断裂感 |
| 术语一致性 | 同一概念全文使用统一术语 |

### 中文润色检查项

加载 `references/chinese-cliche-list.md` 获取完整的中文反面案例库。

核心检查项：

| 检查类别 | 具体内容 |
|---------|--------|
| 翻译腔 | "进行了一个实验"→"开展了实验"、"在这种情况下"→"在此背景下" |
| 书面语 | 确保使用学术书面语而非口语 |
| 过度连接词 | "此外"、"与此同时"、"综上所述"高频出现 → 减少使用 |
| 空洞表达 | "具有重要意义"→具体说明意义、"取得了较好的效果"→报告具体数据 |
| 被动语态 | 中文论文避免过度使用被动句式 |
| 术语一致性 | 全文统一使用同一术语 |
| 段落衔接 | 靠逻辑自然衔接，不过度依赖连接词 |
| 标点符号 | 中文论文使用中文标点，英文文献引用处用英文标点 |

### 中文反面案例库（核心条目）

以下为必须识别和修正的问题模式：

| 问题类型 | 错误模式 | 修正方向 |
|---------|---------|--------|
| 翻译腔动词 | "进行/做出/给予+抽象名词" | 使用具体动词（开展、执行、提供） |
| 模糊限定 | "数据表明某种趋势"（无数据支撑） | 报告具体 p 值和效应量 |
| 空洞意义 | "重要的理论和实践意义" | 具体说明对谁有意义、什么意义 |
| 过度总结 | "综上所述"频繁出现 | 每篇论文最多使用1次 |
| 同义反复 | "进一步深入研究" | 去除冗余，"深入研究"或"进一步分析" |

## 格式规范检查

### 通用格式检查

| 检查项 | 说明 |
|--------|------|
| 引用格式 | 根据引用风格（APA/Vancouver/GB-T-7714）检查格式一致性 |
| 图表编号 | 图表是否有编号，是否按出现顺序编号 |
| 图表引用 | 正文中是否引用了每个图表 |
| 缩写定义 | 每个缩写首次出现时是否定义了全称 |
| 数字格式 | 统一数字格式（如小数点位数） |
| 单位格式 | 统一使用国际单位制（SI） |
| 参考文献列表 | 参考文献列表是否完整、格式是否一致 |

### 中文学位论文特殊检查（GB/T 7713）

| 检查项 | 说明 |
|--------|------|
| 页码格式 | 是否符合学校模板要求 |
| 标题层级 | 一级标题、二级标题格式是否统一 |
| 公式编号 | 公式是否有编号且按章编号 |
| 表格格式 | 是否使用三线表 |
| 参考文献格式 | 是否符合 GB/T 7714 标准 |

## 输出格式

润色建议按以下格式输出：

```
## 润色报告

### 统计摘要
- 总检查处：X 处
- 必须修改：Y 处
- 建议修改：Z 处

### 修改建议

#### 1. [必须/建议] 翻译腔修正
- 位置：第X段，第Y行
- 原文："..."
- 建议："..."
- 理由：...

#### 2. ...
```

## 输入输出

- 输入：待润色的论文文件（全文或指定章节）
- 输出：润色建议报告 + 更新后的文件

## 文件读写约定

| 操作 | 文件 |
|------|------|
| 读取 | `paper/metadata.json`, 待润色的 `paper/draft-*.md` 或 `paper/abstract.md`, `references/chinese-cliche-list.md` |
| 写入 | 更新润色后的对应文件 |
| 更新 | `paper/progress.md`（标记审校润色阶段完成） |

## 注意事项

- 润色前必须先确认润色范围（全文/单章/摘要）
- 不改变论文的核心论点和数据
- 每处修改必须给出理由，不做无理由修改
- 如果 `paper/metadata.json` 不存在，提示用户先加载 `paper-coordinator`

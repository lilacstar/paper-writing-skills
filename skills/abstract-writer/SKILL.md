# Abstract Writer — 摘要与标题生成

本 Skill 根据论文全文生成专业摘要和候选标题。摘要必须忠实反映正文内容，不得夸大或添加正文中没有的信息。

## 前置条件

在执行前，确认以下文件存在：
- `paper/draft-full.md`（合并后的完整论文）或所有分章节文件

如果完整论文不存在，提示用户先使用 `paper-coordinator` 合并全文。

## 工作流程

1. 读取 `paper/metadata.json` 获取论文类型和语言
2. 读取论文全文（`paper/draft-full.md` 或逐个读取 `paper/draft-*.md`）
3. 向用户确认摘要格式：
   - **结构式摘要**（Structured Abstract）：分 Background/Methods/Results/Conclusion 四部分
   - **非结构式摘要**（Unstructured Abstract）：连贯段落
   - 默认根据论文类型推荐：英文期刊论文通常用结构式，中文学位论文通常用非结构式
4. 生成摘要
5. 生成 3-5 个候选标题
6. 输出到 `paper/abstract.md`

## 摘要撰写规范

### 结构式摘要（英文期刊论文推荐）

```
Background: [1-2句] 研究背景和目的
  - 为什么做这个研究
  - 研究的核心问题是什么

Methods: [2-3句] 研究方法
  - 研究设计类型
  - 研究对象/样本
  - 主要干预/暴露措施
  - 主要结局指标和分析方法

Results: [3-5句] 主要结果
  - 样本量
  - 主要发现（附关键数据）
  - 统计显著性

Conclusions: [1-2句] 结论
  - 核心结论（不超出结果范围）
  - 临床/理论意义
```

### 非结构式摘要（中文学位论文推荐）

```
[目的] 1-2句：研究背景和目的
[方法] 2-3句：研究设计、对象、方法
[结果] 3-5句：主要发现和关键数据
[结论] 1-2句：核心结论和意义
```

### 字数控制

| 论文类型 | 建议字数 |
|---------|--------|
| 英文期刊论文 | 150-300 词 |
| 中文期刊论文 | 200-500 字 |
| 中文学位论文 | 500-1000 字 |

## 标题生成规范

生成 3-5 个候选标题，每个标题需满足：

1. **信息密度**：包含核心变量/概念
2. **简洁性**：避免冗余词汇，去除"关于...的研究"等空洞表达
3. **准确性**：不得夸大研究范围或结论
4. **可检索性**：包含有助于数据库检索的关键词

### 标题评估维度

对每个候选标题给出以下评估：

| 维度 | 说明 |
|------|------|
| 信息完整度 | 是否包含研究对象、干预/暴露、结局等核心要素 |
| 关键词覆盖 | 是否包含该领域的常用检索词 |
| 长度 | 是否在合理范围内（英文 8-15 词，中文 15-25 字） |
| 吸引力 | 是否能引起目标读者兴趣 |

### 标题模式参考

**英文**：
- 陈述型："Effect of X on Y in patients with Z: a randomized controlled trial"
- 疑问型："Does X improve Y in Z? A systematic review and meta-analysis"
- 冒号型："X and Y: a prospective cohort study of Z"

**中文**：
- 陈述型："X 对 Y 的影响：一项随机对照试验"
- 描述型："基于 X 方法的 Y 研究"
- 冒号型："X 与 Y 的关系：以 Z 为例"

## 语言规范

### 摘要中的语言约束

- 摘要中不得引用参考文献
- 不得使用缩写（首次出现应写全称），除非是通用缩写（DNA、HIV等）
- 不得包含正文中没有的信息或数据
- 结论部分不得超出 Results 支撑的范围
- 避免使用"首次证明"、"重大突破"、"开创性"等夸大表述

## 输入输出

- 输入：论文全文（`paper/draft-full.md` 或分章节文件）
- 输出：摘要草稿 + 候选标题 → `paper/abstract.md`

## 文件读写约定

| 操作 | 文件 |
|------|------|
| 读取 | `paper/metadata.json`, `paper/draft-full.md`（或所有 `paper/draft-*.md`） |
| 写入 | `paper/abstract.md` |
| 更新 | `paper/progress.md`（标记摘要标题阶段完成） |

## 注意事项

- 如果论文全文不存在，提示用户先合并全文
- 摘要必须基于正文内容生成，不得编造
- 如用户对某个标题不满意，可根据反馈调整
- 生成后建议用户加载 `verify-abstract` 验证摘要与全文的一致性
